# 🧩 Type Challenges

**Link:** [https://github.com/type-challenges/type-challenges](https://github.com/type-challenges/type-challenges)

A popular open-source GitHub repository with TypeScript type-level challenges from beginner to extreme level.

### Best for:

- Deep TypeScript type-system practice
- Solving problems using types only
- Understanding how utility types work internally
- Practicing easy, medium, hard, and extreme challenges

### How to use it:

Clone the repository and solve the challenges locally. Each challenge includes test cases. If the type errors disappear and the tests pass, your solution is correct.

### 🎥 Video Solutions

**Playlist:** [TypeScript Type Challenges Solutions by Dimitri Mitropoulos](https://www.youtube.com/playlist?list=PLOlZuxYbPik180vcJfsAM6xHYLVxrEgHC)  
**GitHub Issue:** [TypeScript Type Challenges Solutions by Dimitri Mitropoulos](https://github.com/type-challenges/type-challenges/issues/21338)

Dimitri Mitropoulos is known for deep TypeScript work, including running Doom inside TypeScript’s type system. His solution videos are very useful because he often explains multiple approaches to the same problem.

### How to use it:

Do not watch the solution video before solving the challenge.

First, give the problem enough time. Think, try different approaches, and solve it on your own as much as possible.

After solving the challenge, you should still watch the solution video. Even if your answer is correct, his explanations can help you understand different ways of thinking about the same problem.

This will make your type-level problem-solving stronger, because you will not just know one solution. You will also understand alternate approaches, trade-offs, and cleaner patterns.

# ✅ Final Recommendation

**Type Challenges is the OG resource** for TypeScript type-level practice. Many other platforms are inspired by its style, and some challenges on other platforms are also ported from Type Challenges.

Completing the **Easy** and **Medium** type-level challenges from **Type Challenges** is non-negotiable. These are the assignments for this section and whatever we have learnt till now in this course.

You should be able to solve them easily using the knowledge we have acquired so far.
